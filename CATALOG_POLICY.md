# WATI Core — Catalog Policy

Version of this policy: 1.1  
Applies to WATI Core v2.0.0 and later unless replaced by a newer policy.

## Purpose

WATI Core is an interoperability registry for Minecraft Bedrock add-ons. It maps identifiers and namespaces to their public source projects and exposes normalized metadata to compatible consumers such as Netbound! and WATI Codex.

## What the public catalog contains

The catalog may contain:

- names of third-party projects and authors for identification and attribution;
- namespaces and identifiers used by Minecraft Bedrock content;
- original WATI fallback names and aliases;
- normalized content classifications;
- normalized functional recipe relationships among identifiers, ingredients, stations and results;
- version and manifest metadata needed to distinguish catalog sources;
- normalized acquisition relationships such as direct block loot, entity drops, cultivation, mining, trading, structure loot, fishing, and natural generation when the analyzed data supports them.

## What WATI Core does not include

WATI Core does not include or redistribute third-party add-ons, original pack files, source code, scripts, textures, models, animations, geometry, user interfaces, audio, documentation, original recipe files, comments, or the original directory and file structure of referenced projects.

Recipe information is stored in WATI's own compact schema as interoperability metadata. It is not a copy of the original recipe JSON file or its formatting.

## Third-party rights

Third-party names, trademarks and identifiers remain the property of their respective owners. A catalog entry does not imply affiliation, sponsorship, endorsement, partnership or official support.

The license shown for a third-party project describes the public project or a notice bundled with the analyzed version. It does not relicense that project, and it does not change the license of WATI's own code, catalog or artwork.

## Corrections and removal requests

Project authors may request:

- correction of a project name, author, namespace, version or license note;
- attribution changes;
- removal of a catalog source or specific entries;
- replacement of an unofficial entry by an official WATI provider.

Requests should be opened in the WATI Core GitHub repository with enough information to verify project ownership. Good-faith requests will be reviewed before the next catalog release.

## Accuracy and legal note

WATI tries to keep identifiers and source attribution accurate, but catalog data can become outdated when third-party add-ons change. Users and creators should verify the referenced project's current page and license.

This file documents project policy and is not legal advice.
